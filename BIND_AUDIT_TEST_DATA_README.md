## ✅ 数据特性

1. **时间跨度合理**：申请时间从 2026-03-01 到 2026-03-09，覆盖最近一周
2. **状态分布真实**：待审核占多数（53%），符合实际业务场景
3. **审核人统一**：所有已审核记录的审核人都是"管理员"（ID: 1900000000000000007）
4. **备注多样化**：包含有备注、无备注、各种类型的备注
5. **数据关联性**：
   - 同一用户可以多次申请绑定不同房屋
   - 同一房屋可以被不同用户申请（包括重复申请被拒的情况）
   - 所有用户和房屋 ID 都来自数据库现有数据

## 🚀 使用方法

在 MySQL 中执行以下命令：

```bash
mysql -u root -p city_ease < insert_bind_audit_test_data.sql
```

或在数据库管理工具中打开并执行 `insert_bind_audit_test_data.sql` 文件。

## 📌 注意事项

1. 执行前请确保数据库中已存在相关用户和房屋数据
2. 如需重复执行，请先删除已插入的数据（脚本中已提供 DELETE 语句注释）
3. 数据 ID 使用前缀 `20260309` 开头，避免与现有数据冲突
4. 所有手机号、姓名与 sys_user 表保持一致
